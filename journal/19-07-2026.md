## Decision

1- Change the user_id s to player_id in the database model definitions.

2- Rename the Enum classes to remove Enum suffix and make them more business value related.

3- Add `composide indexes` to the db schemas.
## Why?

1- It's more accurate and aligns with the domain language.

2- It's more readable and makes the code more maintainable.

3- It will make queries faster and more efficient index means less work for the database.

## Alternatives?

1- Keep the user_id and Enum suffix.

2- Keep Enum suffix.

3- Not adding indexes. or having simple indexes.

## Trade-offs?

1- It's a breaking change and will require a migration.

2- It might not be as good !

4- Wouldn't have been this performant id I use no-index or bad indexes.
