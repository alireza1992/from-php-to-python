## Decision

1- Start with Player and Search domains.

2- Rename User to Player.

3- Change the project structure to be more aligned with DDD.

## Why?

1- They are the most important/core part of the platform.

2- It resonates more with the business values. 

3- Having different domains separated with clear boundaries makes it more maintainable and scalable.

## Alternatives?

1- Starting with match domain.

2- Keep the User domain.

3- Keep the current project structure which is domain-> entities, services etc..

## Trade-offs?

1- Match domain has dependencies related to player and search domains .

2- User is a well established term in the industry, and it's better to keep it for future-proofing.

3- Current structure feels more relatable and familiar with other projects I have worked on.
